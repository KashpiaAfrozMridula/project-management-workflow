Story Status Lifecycle

Detailed lifecycle stages from idea to production deployment.

Draft

Story created by Product/Team Lead, detailing initial requirements.

Pending Story Approval

Awaiting sign-off from key stakeholders (e.g., VP, CPO, CTO, CEO).

Approved

Story is prioritized and ready for technical breakdown and assignment.

Pending UX Design

Ready to enter the design queue.

In UX Design

The Design team is actively creating mockups and flows.

Pending UX Approval

Designs are awaiting for sign off from stakeholder.

Pending Dev

Design Handed over; Task is broken down, estimated, and waiting for a developer to pick it up.

Dev in Progress

Development team is coding the full-stack task and all sub-tasks.

PR in Review

Code is complete and waiting for peer review by a fellow developer/lead.

PR approved

Code review complete; ready to merge to the main integration branch.

In Dev env

Code deployed to the development environment for initial testing.

Pending QA

Code deployed to QA environment and ready to be picked up by the Quality Assurance team.

QA in Progress

The QA team is actively testing the feature against acceptance criteria.

QA Passed
QA Failed > Rework

Pending BV - Stage

QA passed; Deployed to the staging environment for business validation.

BV Passed
BV Failed > Rework

Pending Prod Approval

Waiting for Go-No-Go decision.

Migrate Prod

Deployment to production.

Done

Lifecycle of Dev Tickets (No QA Required)

A streamlined flow for technical tasks or fixes that bypass the testing pipeline.

Pending Dev

Ticket is ready and waiting for a developer to start work.

Dev in Progress

Developer is actively working on the technical task.

PR in Review

Code is submitted for peer review.

PR approved

Code review complete and approved for merging.

In Dev env

Code deployed to development environment for verification.

Done

Task completed and closed directly after dev verification.

Lifecycle of UX Design Tickets

A focused flow for standalone user experience and interface design tasks.

Pending UX Design

Design request is in the backlog, waiting for resource assignment.

In UX Design

Designers are actively creating mockups, wireframes, and prototypes.

Pending UX Approval

Final designs are submitted for stakeholder review and sign-off.

Done

Designs approved and moved to handover or project folder.
